    - 새로운 미지인 것과 새로운 자료가 서로 보다 더 가깝게 되도록 하기 위해서 미지인 것이나 자료 또는 필요하다면 두가지를 다 변형 할 수 있을까?

- 오른쪽이면 현재 노드의 오른쪽 자식으로 이동, 아니면 왼쪽으로 이동

5. 선행 :
- 일단 노드마다 일일이 보내는 건 안됨
- 갱신은 할게 없고, 그냥 몇개 있는지 올려줘야 될 것 같은데
- 올려준다면, 왼쪽 오른쪽 중에 최대 지그재그가 될만한 수를 골라야됨, 근데 이렇게 3이 있고 뭘해야함
  1. 첫번쨰 예시 :  
     - 부모노드에서 3이 반환되야함
     - 최대가 3인걸 알고 있어야하고, 그 최대가 현재 +1이 안되는걸 알고 있어야함 
     - 그니깐 마지막이 right로 끝나서 현재 right에서는 +1할 수 없다는걸 알려줘야함
     - 근데 이러면 record로 올려줘야되는데, 그러고 싶지않긴한데
     - 그렇다면 밑에 2단계까지 depth를 건드는 건 어떤가, 밑에 것 까지 필요하다면
  1-1. depth2로 지그재그 판단
     - 어느 것이 오른쪽에서 왔는지 케이스를 나눠서 max 가능
     - 올려주는 건, 해당 노드에서의 max, max인데 왼쪽에서 max인지, 오른쪽에서 max인지는 잘 생각해봐야함
     - depth2를 어떻게 한다는 거지, 첫번째 노드만으로는 왼쪽 오른쪽인지 몰라서? 그러면 노드 2개로는 가능한가?
     - 아니 근데 맨 밑에도 max를 가져오는데,이건 왼쪽 오른쪽인지 어떻게 알고?
  1-2. 자식 노드에서 최대를 올려주는데, 마지막이 왼쪽 오른쪽인지 확실하지 않아서 +1을 해줘야될지 모를떄
     - 올려줄떄 마지막이 right인지, left인지 구분해서 올려주기
     - 마지막 노드인 경우, 방향이 없으니 null
  1-3. 내려줄때 방향에 따라 왼쪽, 오른쪽 판단해서 +1인지 0으로 초기화 할건지 진행
     - 판단 이유는 자식 노드에서 2개의 값을 올려야함 오른쪽 최대, 왼쪽 최대
     - 계산이 가능할 것 같긴한데, 한 자식이 2개이상 값을 올린다는건 2번 이상 탐색을 한다는거
     - 이러면 N의 횟수가 늘어난다. 이렇게 할거면
       - 부모에서 내릴떄 왼쪽, 오른쪽에 따라서 이어서 count를 해줄건지 아니면 초기화를 해줄건지
       - 거기에서 노드를 방문할떄마다 max를 처리해주는 방식으로 진행하자
